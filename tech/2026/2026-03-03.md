# Notes

## Software Component Discussion

Goal: provide a more general metadata abstraction than a **package**.

Use cases:

- Referencing software at a conceptual level rather than a specific package version.
- Example: referencing **OpenSSL** in general rather than a specific build.

Characteristics:

- Likely not used in **build SBOMs**.
- Useful for **data storage** and abstraction.
- Potentially applicable in **design SBOMs**.

Discussion points:

- Relationship to **hardware component abstractions**.
- Possible equivalence to **libraries**.
- Nicole suggested similarity to hardware abstraction.
- Greg disagreed; discussion continued.

Joshua noted this concept behaves similarly to a **parent class** used for modeling efficiency.

Comparison raised with **NTIA Primary Component**; determined to be different.

Expectation: unlikely to appear in **CVE reports**, since those typically reference specific versions and suppliers.

### Consensus

Rename:

`SoftwareComponent` → `AbstractPackage`

Purpose: clarify modeling intent.

Action:

- Alexios will update the PR.
- PR will be merged afterward.
