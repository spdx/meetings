## Notes
- By 3.1-RC2, every profile should have examples in https://github.com/spdx/spdx-examples
- Existing profiles can add SPDX 3.1-specific examples in `exampleXX/spdx3.1` folders as well.
- Discussion of MIME types for SPDX 3.
- Bob working with OMG to get acceptance for x.y format.
- Kate has reservations with `.spdx3`. Add preference for `.spdx`.
- Discussion that version should be used with extensions.
- Gary feels that it will make it easier for tool developers to have `.spdx3`.
- Joshua feels tools can deduce version; asked what happens when `.rdf` drops in SPDX 4, noting that would not be pleasant to figure out. Feels that extensions with different MIME types should probably work. Knowing how to get the version of a document is important. MIME type needs to be unique.
- Consensus is separate MIME types only for major version changes.
- In this case will be `.json`, not `.jsonld`.
- So this will end up being `.spdx3.json` for the extension.
- Who will be filing for the new extension types with IANA? Art willing to do the effort and forward to IANA. Recommend reaching out to Rose.
